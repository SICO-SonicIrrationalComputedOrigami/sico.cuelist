# sico.cuelist
shape with the style you like the electronic score in Max with JS

The main idea of this approach, is to allow the Composers and the Computer Music Designers to design an electronic score sufficently agnostic to the programming software they're using, detached and therefore more easy to maintain. Moreover, the goal is: to have a readable score in its completess, to be indipendent from external libraries (meaning to be less affected by obsolescence problems) and to open to the option of using external code editors which offer more refined features.
